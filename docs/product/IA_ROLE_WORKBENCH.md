# 角色工作台与信息架构契约

> 状态：权威 IA v1.0 · 2026-07-16

## 1. 导航原则

信息架构围绕用户任务，而不是后端模块。登录后统一进入 `/dashboard`，同一路由按角色提供不同工作台；“上传、我的文件、审核、系统治理”保持稳定一级入口。详情页是上下文页，不进入常驻导航。

```text
公开
├─ 登录 / 注册
├─ 邮箱验证 / 找回密码
└─ 服务状态与帮助

登录后
├─ 工作台 /dashboard（角色自适应）
├─ 文档
│  ├─ 上传 /upload
│  ├─ 我的文件 /my-files
│  └─ 审核队列 /files（管理员）
├─ 知识库治理（系统管理员）
│  ├─ 分类 /categories
│  ├─ 标签 /tags
│  └─ Dataset /datasets
├─ 运营与运维
│  ├─ 同步任务 /task-logs（管理员）
│  ├─ 统计 /statistics（系统管理员）
│  └─ 审计 /audit-logs（系统管理员）
└─ 系统
   ├─ 用户 /users
   ├─ 部门 /departments
   ├─ AI /ai-config
   ├─ 设置 /settings
   └─ 个人中心 /profile
```

## 2. 员工工作台

首屏回答三件事：“我现在要做什么、最近上传到哪一步、哪里需要我处理”。

- 主操作：上传文档。
- 待办：草稿待提交、被驳回待修改、敏感复核待确认。
- 进度：待审核、处理中、已同步、失败；点击带当前筛选跳转“我的文件”。
- 最近活动：最近 5 条文档和通知，显示下一步而不只显示技术状态。
- 门禁卡：未验证邮箱或未分配部门时替代业务卡片，并给唯一恢复动作。

## 3. 部门管理员工作台

首屏回答：“队列是否失控、我应该先审哪份、同步是否异常”。

- KPI：未领取、我领取、2 小时内到期、已超 SLA、同步失败。
- 优先队列：已超时 > critical/high 敏感 > 临近 SLA > 最早提交。
- 快捷操作：领取下一份；继续我的审核；查看失败任务。
- 团队负载：每位审核人领取数与超时数，仅展示管辖范围。
- 审批详情采用左侧原件/分析，右侧元数据/决定的双栏；窄屏单列且底部固定操作条。

## 4. 系统管理员工作台

- 系统健康：API、数据库、RabbitMQ、Redis、MinIO、worker 心跳、DLQ。
- 流程健康：待审年龄分布、同步成功率、各状态积压、失败类别。
- 治理：未分配用户、无有效 Dataset 映射的部门、临期/过期文档。
- 资源：MinIO 容量、月新增、LLM token/估算成本、RAGFlow 调用量。
- 任何红色指标必须有可执行的下钻路径，不能做没有去向的装饰 KPI。

## 5. 列表与搜索

所有列表采用服务端分页。通用查询参数：`page`（默认 1）、`page_size`（默认 20，最大 100）、`q`、`sort`、`order`。响应统一包含 `items`、`page`、`page_size`、`total`、`total_pages`。

文档搜索覆盖文件名、标题、提交人和可公开标签，不搜索原文；部门管理员查询始终追加服务端数据域。筛选与页码保存在 URL；P2 保存视图只保存查询定义和列偏好，不保存结果数据。

## 6. 移动端

- ≤768px：桌面侧栏移为抽屉，顶栏保留菜单、标题、通知和头像；正文单列。
- 表格优先转信息卡；确需横向滚动时显示提示并固定主标识列。
- 审批可浏览原件、填写理由和确认决定；复杂 Dataset/系统配置在手机上允许只读并提示桌面完成。
- 断网、上传中离开和后台恢复必须有明确反馈，不用 hover 承载信息。

## 7. 通知深链白名单

允许的内部目标仅包括 `/dashboard`、`/my-files`、`/files/:id`、`/task-logs` 与 `/profile`。后端保存结构化 `resource_type/resource_id`，前端生成路由；拒绝存储或跳转任意外部 URL。权限变化后打开深链统一落无权限/已失效页。
