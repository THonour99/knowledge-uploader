## 3. 计量、价格与异常语义

以下项目必须形成确定规则，不能由实现人员自行猜测：

- token、请求次数、缓存命中、批处理折扣与其他计费单位的权威来源；
- 价格表版本、生效时间、历史价格回溯和模型别名映射；
- Provider 未返回 usage、返回部分 usage、价格未知或旧数据缺失时的状态语义；
- known、estimated、unknown、unsupported、stale、unavailable 的展示与聚合规则；
- 重试、超时、取消、流式中断、幂等重放和 Provider 已扣费但本地失败时的计量规则；
- 预算检查与任务入队之间的并发竞争、预留超时及补偿规则；
- 超额后的拒绝、降级、排队或人工审批策略；
- 管理员临时豁免的授权范围、有效期、双人复核和审计要求；
- 成本数据的最小保留期、访问范围、导出脱敏与删除规则。

## 5. 实施影响清单

最终决策批准后，实施 PR 必须逐项确认；不适用项也要写明理由。

- [ ] 配置契约：键名、类型、默认值、密文规则、热更新行为和环境覆盖关系。
- [ ] 数据库与迁移：预算、预留、结算、价格版本、告警和豁免记录；升级/降级验证。
- [ ] API：查询、修改、预检、拒绝和错误码；仅向有权角色开放。
- [ ] 任务链：入队前原子预留、幂等键、重试结算、超时释放和补偿任务。
- [ ] 前端：预算进度、口径、数据新鲜度、未知成本、阻断原因和有权操作。
- [ ] 指标告警：消耗率、剩余额度、未知 usage、预留泄漏、拒绝次数和结算差异。
- [ ] 审计与隐私：不记录 API Key、文档内容或无关个人数据。
- [ ] 运维手册：价格更新、应急关闭、超额处置、差异对账、恢复和回滚。
- [ ] 离线测试：边界值、并发、跨周期、重试、未知价格、Provider 异常和权限隔离。
- [ ] 真实 Provider 验证：使用专门测试预算与最小调用量，证据中不包含密钥。

## 6. 发布验收证据

只有以下证据全部齐备，才可以申请把 COST-002 从“未决”改为最终状态：

1. 产品、财务、安全和平台责任人对同一版本决策表完成签署；
2. 决策对应的配置、迁移、API、前端、告警、审计和运维手册已合并；
3. 自动化测试覆盖预算边界、并发竞争、重试、未知价格和权限隔离；
4. 在隔离测试预算下完成最小真实 Provider 验证，并保存账单/usage 对账证据；
5. 完成应急关闭与恢复演练，证明不会在阻断后继续产生未授权付费调用；
