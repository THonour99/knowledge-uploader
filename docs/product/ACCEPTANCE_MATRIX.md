## 发布证据模板

每次候选发布在 `docs/phase-reports/<version>/` 保存：commit SHA、镜像 digest、迁移前后 revision、执行环境架构、测试命令与原始日志、失败/重试记录、ARM64 设备信息、备份恢复时间线、告警截图/事件 id、未关闭豁免。任何密钥在归档前脱敏。

## 最低发布判定

- P0 全部通过；P1 中 `CFG/E2E/DLQ/OBS/ARM/DR` 不允许豁免。
- 单元测试通过不能替代真实基础设施 E2E；buildx 能构建不能替代 ARM64 实机运行。
- 浏览器全 mock E2E 只能作为 UI 回归，不能作为主链证据。
