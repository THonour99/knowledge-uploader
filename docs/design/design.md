# Knowledge Uploader 产品视觉与工作台设计

> 版本：2.0 · 2026-07-16
>
> 本文是页面布局与产品交互的视觉权威源。基础色阶、字体、控件尺寸和原型参考 [设计系统规范](../design-system.md) 与 [主应用原型](../design-system/prototype-app.html)；业务状态和动作以 [05 状态/API 契约](../../需求文档/05_DATABASE_API_SPEC_数据库与API规范.md) 为准。

## 1. 设计方向

主题采用 Emerald（知识、成长、主操作）+ Stone（温暖、长期阅读）。界面应像可靠的内部工作工具：层级清楚、动作具体、信息密度适中。禁止用渐变大标题、装饰图形、无去向 KPI 和大量同构卡片制造“看板感”。

设计优先级：下一步动作 > 风险/时效 > 当前状态 > 统计。每屏只保留一个视觉主操作；颜色是辅助，文案与图标必须同样能说明状态。

## 2. Token 迁移契约

`frontend/src/theme/tokens.ts` 是代码单一源，必须从当前 Blue/Gray 迁移到下表；组件不得继续硬编码旧蓝色。迁移需一次完成 token 与语义映射，再逐页替换，避免一半蓝、一半绿。

| 语义 | Light | 用途 |
|---|---|---|
| Brand | `#059669` | 主操作、链接、选中 |
| Brand hover | `#047857` | hover/active |
| Brand soft | `#D1FAE5` | 选中浅底 |
| Base | `#FAFAF9` | 页面底 |
| Surface | `#FFFFFF` | 卡片/浮层 |
| Subtle | `#F5F5F4` | 表头/次级底 |
| Text | `#1C1917` | 主文字 |
| Text secondary | `#57534E` | 次文字 |
| Border | `#E7E5E4` | 常规边框 |
| Success | `#16A34A` | 已通过/已同步，不代替 Brand |
| Warning | `#D97706` | 待审/临期 |
| Danger | `#DC2626` | 驳回/超时/敏感/失败 |
| Info | `#2563EB` | 处理中 |

圆角：控件 8、卡片 12、状态胶囊 999；间距以 4px 阶梯；正文 14/22。完整值不在本文复制，直接复用设计系统规范。

## 3. 应用外壳

桌面：220px 侧栏 + 56px 顶栏 + 最大 1440px 内容区。侧栏按“工作台 / 文档 / 知识库 / 运营 / 系统”分组；顶栏只放移动菜单、当前页、全局搜索（有真实消费者后）、通知和用户。

公告使用独立图标与右侧抽屉，不复用业务通知角标。最新一条未读、有效、置顶公告显示在顶栏下方的全局置顶条；关闭或进入详情即标记已读。公告正文只在懒加载的详情/管理页面渲染 Markdown，抽屉和置顶条仅展示标题。

详情页不常驻侧栏。返回列表必须保留筛选和页码。通知、状态数字和错误指标都必须有可访问的下钻目标。

## 4. 员工工作台

员工首页不用四到六张 KPI 卡铺满首屏，采用三个连续工作区域：

```text
┌ 页面标题：上午好，张伟                     [上传文档] ┐
├ 开放状态轨道 Open status rail                         ┤
│ 草稿 3 ── 待审核 2 ── 处理中 1 ── 已入库 18          │
│             被驳回 1（危险分支，直接可处理）           │
├ 继续处理                                                ┤
│ [驳回] 客户名单.xlsx  查看原因 / 修改并重提             │
│ [草稿] 员工手册.docx  补充信息 / 提交审核                │
├ 最近文档（真实表格：文件、下一步、状态、更新时间）       ┤
└ 通知/帮助作为右侧或末尾次级内容                          ┘
```

开放状态轨道是一条可点击的流程摘要，不是互不相干的 KPI 卡。数字来自同一服务端筛选口径，点击进入 `/my-files` 对应状态。处理中必须区分 AI 与 RAGFlow，但不让用户理解内部队列名。

“继续处理”只显示员工能行动的草稿、驳回和敏感复核；每项只有一个推荐动作。最近文档用表格保持可扫描性，不重复堆卡。

## 5. 部门管理员工作台

管理员首页以审核队列为主体，不使用 KPI 卡阵列：

```text
┌ 文件审核                           [领取下一份] ┐
├ 队列分段：未领取 12 | 我的 4 | 临近 SLA 3 | 超时 1 ┤
├ 筛选/搜索/排序（URL 持久化）                       ┤
├ 审核表格                                              ┤
│ 风险 | 文件/提交人 | 等待时长/SLA | 领取人 | 动作     │
└ 右侧抽屉/详情：原件预览 + AI/敏感 + 元数据 + 时间线  ┘
```

排序默认“超时 > critical/high > 临近 SLA > 最早提交”。领取后动作变为“继续审核”，被他人领取的行显示姓名和时间但不暴露无关信息。领取冲突用行内反馈和刷新，不用泛化 toast 覆盖上下文。

审核详情桌面采用左 2/3 原件预览、右 1/3 决策面板。决策面板固定显示：风险、分类、Dataset、理由以及必选“批准并同步 / 仅批准不入库”；拒绝是独立危险动作。原件加载失败仍提供鉴权下载。

## 6. 系统管理员工作台

系统管理员先看异常与可执行下钻：依赖健康、outbox/队列/DLQ、超 SLA、同步失败、未分配用户、无有效映射部门、容量告警。可用少量数字，但每项呈现在分组状态面板或趋势表中，禁止再造装饰 KPI 墙。

容量/成本未实现前不展示 `0` 假数据；使用明确的“尚未接入指标”占位和建设状态。

## 7. 状态映射

| 技术状态 | 用户文案 | 语义色 |
|---|---|---|
| `uploaded` | 草稿 | neutral |
| `extracting_text/analysis_queued/analyzing` | AI 处理中 | info |
| `analyzed` | 分析完成·待提交 | info |
| `analysis_failed` | AI 分析失败 | danger |
| `sensitive_review_required` | 敏感复核 | danger |
| `pending_review` | 待审核 | warning |
| `approved` | 已批准·未入库 | success |
| `rejected` | 已驳回 | danger |
| `queued/syncing/uploaded_to_ragflow/parsing` | 入库处理中 | info |
| `parsed` | 已入库 | success |
| `failed/ragflow_cleanup_failed` | 处理失败 | danger |
| `disabled/deleted` | 已归档/已删除 | neutral |

同一页面不要同时用“文件状态/审核状态/同步状态”三个含混标签。列表展示一个主文案和必要的次级说明，详情再拆维度。

## 8. 移动端

- ≤768px 侧栏变左侧 drawer；顶栏菜单 44×44px，正文单列。认证页只显示表单与紧凑品牌，不保留横向品牌面板。
- 员工状态轨道可横向滚动并有位置提示；继续处理为列表；最近文档转信息卡或明确横向表格。
- 审核队列为紧凑卡片，SLA 与风险先于次要元数据；详情单列，底部 sticky actions 显示“拒绝 / 批准”，打开决策 sheet 后再确认同步分支。
- 主要动作不依赖 hover；弹窗/sheet 不越出视口，支持键盘与屏幕阅读器，遵循 reduced motion。

## 9. 原型与图像概念的权威关系

[prototype-app.html](../design-system/prototype-app.html) 保留为色彩、控件密度和主导航参考。原型中的旧蓝色、KPI 卡堆叠、`uploaded` 文案和可空 Dataset 审批若与本文冲突，必须按本文改造，不能以“原型已有”为由保留。

配套 image concept 因生成服务暂不可用，标记为非阻塞待补验收项；它只能验证情绪/构图，不能替代 token、IA、状态/API 和可访问性契约。实现验收以真实页面截图、360/768/1440px 视口和主链交互为准。

## 10. 视觉验收清单

- tokens 已统一 Emerald/Stone，无旧主蓝硬编码；对比度满足 WCAG AA。
- 员工页存在开放状态轨道、继续处理、最近文档表；管理员页以审核队列/SLA/领取/预览为主体。
- 首屏无 KPI 卡墙、无装饰渐变、无没有下钻的数字。
- 360/390/768/1440px 无横向页面溢出；drawer、bottom actions、焦点和键盘顺序可用。
- loading/empty/error/403/404/冲突/失效通知都有独立状态。
