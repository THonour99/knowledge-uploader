# 04. 前端开发与交互规范

> 权威产品结构见 [角色工作台与 IA](../docs/product/IA_ROLE_WORKBENCH.md)，视觉 token 见 [design.md](../docs/design/design.md)。文件状态与 HTTP 字段以 [05 状态机/API 契约](./05_DATABASE_API_SPEC_数据库与API规范.md) 为准；本文只规定实现边界和跨页面行为。

## 1. 技术边界

- React 18 + TypeScript strict；服务端状态统一 TanStack Query，登录与纯 UI 状态统一 Zustand。
- HTTP 调用只走 `src/api/client.ts`，统一 JWT、request id、错误码和 401 处理；前端不得直连 MinIO、RAGFlow 或 AI Provider。
- 状态展示只走 `StatusTag` 与中央状态映射；颜色、圆角、间距只走 `theme/tokens.ts`。
- 路由与菜单共用一份角色元数据。隐藏菜单不能替代路由守卫，路由守卫不能替代后端权限。

## 2. 路由契约

| 路由 | 角色 | 页面 |
|---|---|---|
| `/login`、`/register`、`/forgot-password` | 公开 | 认证入口 |
| `/verify-email?token=`、`/reset-password/:token` | 公开 | 邮件动作落地页 |
| `/dashboard` | 全部登录用户 | 按角色渲染工作台 |
| `/upload`、`/my-files`、`/files/:id` | 登录用户 | 上传、个人列表、受权限详情 |
| `/files` | 部门/系统管理员 | 审核队列与领取 |
| `/task-logs` | 部门/系统管理员 | RAGFlow 任务与失败恢复 |
| `/datasets`、`/ai-config`、`/users`、`/departments`、`/settings` | 系统管理员 | 全局治理 |
| `/statistics`、`/audit-logs` | 系统管理员 | 指标与审计 |
| `/categories`、`/tags` | 系统管理员 | 分类与标签治理 |
| `/profile` | 全部登录用户 | 个人中心与密码修改 |

登录成功后统一跳 `/dashboard`。无权访问返回明确 403 页面；资源不存在或数据域外都显示相同 404 文案，避免存在性泄露。

## 3. 页面状态

每个查询页面必须独立处理：首次加载骨架、刷新保留旧数据、空结果、无权限、网络错误与可重试失败。禁止把空数组同时当“加载中”和“没有数据”。破坏性或不可逆动作必须二次确认并解释影响对象。

列表筛选写入 URL query：`page`、`page_size`、`q`、`status`、排序和页面特有筛选。返回列表时恢复上下文；换筛选时回到第 1 页。

## 4. 主链交互

- 上传页必须显示部门、文件限制、是否自动提交；未验证或未分配部门时禁用选择器并提供恢复入口。
- `uploaded` 标签文案为“草稿”，同时可在详情展示技术状态“已上传”；避免“已上传”和“已提交”混淆。
- 自动提交开启且 AI 运行时，进度必须说明“分析完成后自动提交”，不能在分析中显示“待审核”。
- 审批弹窗用必选单选项“批准并同步 / 仅批准不入库”。选择同步后 Dataset 必填；选择仅批准时隐藏 Dataset 并解释后续可手工同步。
- 原件预览失败时保留下载动作；下载使用鉴权请求和 blob，不把 token 拼进 URL。

## 5. 审核、SLA 与通知

- 审核队列默认分栏：未领取、我领取、临近 SLA、已超时；领取冲突显示当前领取人和刷新动作。
- 详情离开前若表单有未保存元数据必须提示。审批成功后从当前队列移除并保留下一条入口。
- 顶栏通知显示服务端未读数；单条点击先校验深链再标已读，支持“全部已读”。通知失效时仍可标已读。

## 6. 响应式与可访问性

- 断点以 `theme/tokens.ts` 为单一源；≤768px 侧栏变抽屉，认证页单列，筛选折叠，表格允许卡片模式或标注横向滚动。
- 所有功能可键盘完成；焦点可见；图标按钮有可读标签；颜色不是唯一状态线索。
- 触控目标最小 44×44px；弹窗不得超出视口；遵循 `prefers-reduced-motion`。

## 7. 前端验收门禁

- Vitest + Testing Library 覆盖角色路由、状态映射、分页 URL、审批分支、通知闭环和移动导航。
- 构建无 TypeScript 错误；关键路由按页懒加载，单个非必要 vendor chunk 超预算必须解释或拆分。
- 浏览器 E2E 不允许全量 mock API；至少一套走真实后端与真实基础设施，外部 RAGFlow/LLM 可使用协议级受控替身。
